Incus Client

The native client for Incus (which Malleus is using behind the scenes) provides CLI access to manage and control the containers and VMs in your labs.

1. Download Incus Client

Download the Incus client (not agent!) for your operating system. Check the Files page.

2. Register a Remote

With the Incus client, register a "remote" to access the server. When prompted for a token, use the token below. 
incus[.exe] remote add <NAME> <SERVER_IP>
If the server is at "192.168.9.10": 
incus[.exe] remote add malleus 192.168.9.10
Then set it to be the default remote: 
incus[.exe] remote set-default <NAME>
{{ user_key }}

3. Select Project

Each lab is a an Incus project, list projects available to you with: 
incus[.exe] project list
Switch to the project to manage and access the lab's systems. A lab project's name is <USERNAME>-<LAB-NAME> 
incus[.exe] project switch <PROJECT>

4. List Systems

List the systems, referred to as "instances" in Incus, within the project with: 
incus[.exe] list

5. Access Console

You can access a terminal for an instance with: 
incus[.exe] exec <INSTANCE-NAME> -- bash -i
You could replace "bash -i" with any command in the instance.
Note that for Incus clients on Windows, certain terminal functionality may not work! For Windows, it is recommended to use any SSH connections into the lab instead.

SSH - Putty

In the main initial menu, set "Host Name or IP Address" to the IP of the Malleus server, set the port to one given in the lab.
Click "Open" and provide the username and password when prompted.

SOCKS Proxy

Before clicking "Open", go to Connection > SSH > Tunnels.
  1. Ensure "Local ports accept connections from other hosts" and "Remote ports for the same" are checked.
  2. In "Add new forwarded port" put a port (like 8888) in "Source port" and set "Destination" to "Dynamic" without putting anything in the "Destination" textbox.
  3. Click "Add" to insert the port.
Click "Open" and provide the username and password when prompted.

SSH - OpenSSH/Linux

With OpenSSH on Linux, SSH access is simple, with "PORT" the port given in the lab: 
ssh -p <PORT> <USER>@<MALLEUS_IP>

SOCKS Proxy

Using a proxy via SSH can let you access ports internal to the lab networks. OpenSSH has built in support for SOCKS proxies to do this. All you need to do is add the following with you ssh command: 
-D <PORT>
Something like "8888" is recommended. Use this port when configuring applications to use the proxy.
